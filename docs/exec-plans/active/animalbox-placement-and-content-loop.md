# AnimalBox Placement And Content Loop

## Problem

AnimalBox needs to feel like a polished mobile avatar garden, not a static mock. The current biggest risk is that generated assets look acceptable alone but fail once placed on the island: position, scale, tap cells, and UI panels can drift away from the actual background art.

## User Outcome

The player should be able to open the mobile prototype, see a degu that fits the island scale, place decor on believable ground positions without clipping UI or the degu, and keep unlocking/using more content.

## Scope

- Improve decor placement and scale through rendered evidence loops.
- Add missing QA that catches visual placement failures before asset expansion.
- Add small game/content improvements only when they support the mobile garden loop.
- Keep paid gacha out of scope.
- Keep runtime art sourced from ImageGen-approved assets; code can add manifests, placement metadata, and UI around them.

## Non-Goals

- Do not return to DragonBones/rig work in this loop.
- Do not replace the whole renderer with Phaser/Pixi yet.
- Do not add paid purchases, external accounts, or production databases.

## Acceptance Criteria

- [ ] Placement QA covers at least 3 mobile viewports.
- [ ] Placement QA covers all available background themes.
- [ ] QA fails if any selected decor ghost overlaps the placement sheet, bottom nav, or degu keepout beyond the allowed threshold.
- [ ] QA records per-background/per-viewport metrics and screenshots under `output/playwright`.
- [ ] `npm test` passes.
- [ ] `npm run build` passes.
- [ ] A Cloudflare Pages deployment is produced at each stable milestone.
- [ ] Any new runtime assets are generated by ImageGen or promoted from existing ImageGen outputs with provenance.

## Implementation Plan

1. Strengthen `tools/qa/placement-fit-qa.mjs` so it runs a matrix over viewports and background themes.
2. Add placement metrics for valid cell count, ghost-to-panel clearance, degu size ratio, and background ground-mask bounds.
3. Use failures to tune `sceneLayout` or introduce background-specific layout profiles.
4. Add the next highest-value missing game/content feature after QA is stable.
5. Commit, push, deploy, and verify the preview.

## Delegated Lanes

- `Trace the 3rd`: read-only feature gap exploration.
- `Stress the 3rd`: read-only placement/viewport edge-case exploration.

## Stop Conditions

Stop and ask before destructive file cleanup, credentials, paid services, production data changes, external account flows, or a major renderer/framework migration.
